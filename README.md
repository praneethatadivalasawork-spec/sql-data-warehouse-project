# SQL Data Warehouse Project

## Overview
This project focuses on building a **modern data warehouse** using SQL Server to consolidate sales data from multiple sources. The goal is to enable **analytical reporting** and support **data-driven decision making**.

## Objectives
- Develop a scalable and reliable data warehouse.
- Integrate and cleanse data from multiple source systems.
- Provide a user-friendly data model for analytical queries.
- Deliver documentation for transparency and reproducibility.

## Project Scope
- **Data Sources**: Import sales data from two CSV files.
- **Data Quality**: Cleanse and resolve inconsistencies before analysis.
- **Integration**: Combine both sources into a unified model optimized for analytics.
- **Focus**: Work with the latest data only (no historical data required).
- **Documentation**: Provide clear instructions and explanations in this README.

## Key Components
1. **ETL Pre-Processing**
   - Extract data from CSV files.
   - Transform and cleanse data to ensure accuracy.
   - Load into SQL Server staging tables.

2. **Data Modeling**
   - Design a star schema or snowflake schema for analytical queries.
   - Create fact and dimension tables for sales reporting.

3. **Analytics**
   - Enable reporting queries for KPIs such as revenue, product performance, and regional sales.
   - Optimize queries for performance and scalability.
